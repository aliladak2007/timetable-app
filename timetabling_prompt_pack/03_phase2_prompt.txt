Phase 2 only.

Build the admin frontend for the timetabling assistant on top of the existing backend.

Scope:
- teacher management screens
- student management screens
- session list and schedule views
- add new student flow
- match suggestions screen
- confirm booking flow
- conflict warnings and validation messages

Requirements:
- Use React or Next.js
- Keep the UI simple, clean, and practical
- Prioritise speed of use for an admin doing manual scheduling
- Do not add unnecessary animations or decorative elements

Expected screens:
1. Dashboard
2. Teachers page
3. Teacher detail page with weekly availability
4. Students page
5. Student detail page
6. Sessions page
7. Add new student wizard
8. Match suggestions page
9. Confirm booking modal or page

Important:
- Reuse backend APIs rather than duplicating logic in the frontend
- Keep forms validated
- Make time slot selection clear and low-friction
- Add a minimal timetable or calendar-like weekly display if practical
- Show why a slot is valid or invalid where useful
