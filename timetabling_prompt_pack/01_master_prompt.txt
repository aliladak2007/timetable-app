You are a senior full-stack engineer and product architect.

I want you to design and build an MVP for a timetabling assistant app for a tutoring centre. This is not a generic school timetable generator. It is an operational scheduling tool for assigning tutors to students and quickly finding valid slots for new students.

First, understand the product properly, then create a plan, then implement in small reviewed steps.

PROJECT GOAL
Build an app that helps me:
1. Store teacher availability
2. Store student existing timetables
3. Assign teachers to students
4. When a new student is added, input their preferred times
5. Match those preferred times against a specific teacher’s availability
6. Exclude clashes with the teacher’s existing bookings
7. Exclude clashes with the student’s existing commitments
8. Return the best valid timetable options quickly
9. Make the process significantly faster than manually checking Excel sheets

CORE USER WORKFLOW
The app should support this real-world workflow:
- I create teacher profiles
- For each teacher, I enter available time slots
- I create student profiles
- For each student, I store current timetable or unavailable times
- I assign recurring sessions between teachers and students
- When I want to add a new student:
  - I choose a teacher or shortlist of teachers
  - I enter the student’s preferred times
  - the system suggests valid matching slots
  - I can confirm one and create the booking

PRODUCT REQUIREMENTS
Please design for a tutoring business, not a university timetable generator.

The system should support:
- Multiple teachers
- Multiple students
- Weekly recurring schedules
- Teacher availability by day and time
- Student unavailable times or existing commitments
- Existing booked sessions
- Matching a student with a specific teacher
- Optional support for trying multiple teachers and ranking results
- Manual override by the admin
- Clear conflict detection
- Easy editing and deleting of sessions

MVP FEATURES
Implement these first:

2. Student management

   - rank matches sensibly

5. Admin view

   - see all student assignments

MATCHING LOGIC
The matching logic is central.

A slot is valid if:
- it falls inside the teacher’s available time
- it falls inside the student’s preferred time or allowed time
- it does not overlap any existing booked session for that teacher
- it does not overlap any blocked or booked time for that student
- it satisfies the requested lesson duration

Please design this carefully and keep it easy to reason about.

RANKING LOGIC
For suggested slots, rank options using a transparent scoring system.
For MVP, use something simple like:
- exact match to preferred time scores higher
- earlier available slot scores slightly higher
- lower teacher fragmentation is better
- consistent weekly slot is better
- optionally prefer slots that keep the teacher’s timetable compact

Explain the ranking logic in code comments and documentation.
